=========================================================
SIGNED VS. UNSIGNED BINARY NUMBERS (Quick Revision)
=========================================================
A computer's memory just holds 1s and 0s. How those bits 
are translated into decimal depends entirely on whether 
the language reads them as UNSIGNED or SIGNED.

Let's look at 8 bits:  1 1 1 1  1 1 0 1

---------------------------------------------------------
1. UNSIGNED (Purely Positive)
---------------------------------------------------------
- Used for: Colors, IP addresses, memory addresses.
- Rule: Every bit represents a standard positive power of 2.
- Range (8-bit): 0 to 255

Calculation for 1111 1101:
  128 + 64 + 32 + 16 + 8 + 4 + 0 + 1 = 253

---------------------------------------------------------
2. SIGNED (Two's Complement) -> *Default in Java*
---------------------------------------------------------
- Used for: Standard math, standard variables (int, byte).
- Rule: The leftmost bit (Most Significant Bit) has a 
  NEGATIVE weight. All other bits are positive.
- Range (8-bit): -128 to 127

Calculation for 1111 1101:
  Leftmost bit is 1, so it is worth -128.
  -128 + 64 + 32 + 16 + 8 + 4 + 0 + 1 = -3

---------------------------------------------------------
SUMMARY
---------------------------------------------------------
The bits [1111 1101] are exactly the same in memory. 
- If the computer asks "What is the unsigned value?" -> 253
- If the computer asks "What is the signed value?"   -> -3
